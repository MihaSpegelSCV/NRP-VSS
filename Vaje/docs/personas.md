**Product Backlog – Shieldy**

**User Stories**

🔴 **MUST HAVE**

1. As a user,
I want to scan a suspicious link,
so that I know if it is safe.

Acceptance Criteria:

User can input a URL
System returns SAFE or DANGER result


2. As a user,
I want to get a clear warning if a link is dangerous,
so that I avoid scams.

Acceptance Criteria:

Dangerous link shows red alert
Warning message is displayed


3. As a user,
I want to understand why something is dangerous,
so that I can learn to recognize threats.

Acceptance Criteria:

Explanation is shown
At least one reason is provided


4. As a user,
I want step-by-step instructions after detecting phishing,
so that I know what to do next.

Acceptance Criteria:

Minimum 3 steps shown
Steps are clear and actionable


5. As a user,
I want to paste a message for analysis,
so that I can check SMS or email scams.

Acceptance Criteria:

User can paste text
System analyzes content


6. As a user,
I want to see a safety score,
so that I understand my risk level.

Acceptance Criteria:

Score is displayed (0–100)
Score updates after scans


7. As a user,
I want to see if a domain is verified,
so that I trust safe websites.

Acceptance Criteria:

Verified domain label shown
Explanation displayed


8. As a user,
I want to see my scan history,
so that I can track past threats.

Acceptance Criteria:

List of past scans displayed
Each item shows status
🟡 **SHOULD HAVE**


9. As a user,
I want to get a daily security tip,
so that I improve my awareness.

Acceptance Criteria:

Tip displayed daily
User can mark as read
10

As a user,
I want to report a phishing link,
so that others are protected.

Acceptance Criteria:

Report button available
Link is submitted


11. As a user,
I want to see recent scan activity,
so that I stay informed.

Acceptance Criteria:

Recent scans shown on dashboard


12. As a user,
I want to quickly access scan function,
so that I save time.

Acceptance Criteria:

Scan button visible on home screen


13. As a user,
I want to open safe links directly,
so that I continue browsing.

Acceptance Criteria:

“Open link” button available


14. As a user,
I want to learn from past mistakes,
so that I avoid future scams.

Acceptance Criteria:

Past phishing explanations stored
🟢 **COULD HAVE**


15. As a user,
I want a security score history,
so that I track improvement.

Acceptance Criteria:

Historical scores visible


16. As a user,
I want notifications about threats,
so that I react quickly.

Acceptance Criteria:

Notifications can be enabled


17. As a user,
I want gamification (XP/levels),
so that I stay motivated.

Acceptance Criteria:

XP system exists
Level shown


18. As a user,
I want personalized tips,
so that they match my behavior.

Acceptance Criteria:

Tips change based on usage
⚪ **WON’T HAVE ( trenutno )**


19. As a user,
I want integration with banks,
so that I get real-time fraud alerts.

Acceptance Criteria:

Defined but not implemented

I want enterprise-level security tools,
so that companies can use the app.

Acceptance Criteria:

Feature defined for future

📊 Prioritization Method – MoSCoW
Zakaj MoSCoW?

Izbrana je metoda MoSCoW, ker:

omogoča jasno ločevanje med nujnimi in dodatnimi funkcijami
je primerna za MVP razvoj
pomaga fokusirati razvoj na ključne vrednosti za uporabnika
Razlaga kategorij
MUST → nujno za delovanje produkta
SHOULD → pomembno, ampak ni kritično
COULD → dodatna vrednost
WON’T → prihodnost
